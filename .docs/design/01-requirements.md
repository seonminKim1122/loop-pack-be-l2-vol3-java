## 요구사항 정의서
- 기획자, 프론트 개발자, 백엔드 개발자 모두가 함께 작성하는 문서입니다.
- Actor, Conditions, Main Flow, Exception Flow, After 구조로 문서를 작성합니다.
    - Actor: 누가
    - Conditions: 필요조건
    - Main Flow: 메인 흐름
    - Exception Flow: 예외 흐름
    - After: 후속 흐름


※ 이 문서에서 Guest(비회원), User(회원), Admin(관리자) 를 의미합니다.

## 회원
※ 회원에 대한 인증은 아이디, 비밀번호가 올바른지를 확인하는 과정을 의미<br>
※ 로그인은 아이디, 비밀번호를 입력해 인증 하는 과정<br>
※ 인증은 요청 헤더의 아이디, 비밀번호로 확인

### 1. 회원가입
- **Actor**: Guest
- **Conditions**
  - 아이디, 비밀번호, 이름, 생년월일, 이메일 정보는 필수
  - 아이디
    - 5 ~ 20자의 영문대소문자, 숫자만 가능
  - <a id="signup-pw"></a>비밀번호
    - 8 ~ 16자의 영문대소문자, 숫자, 특수문자만 가능
    - 허용 특수문자( '!', '@', '#', '$', '%', '^', '&', '*', '(', ')' )
    - 비밀번호에는 생년월일(yyyyMMdd 전체 문자열) 포함 불가능
    - 비밀번호는 반드시 암호화해서 저장
  - 이름: 2 ~ 6 자의 한글
  - 생년월일
    - 'yyyyMMdd' 형식을 따라야 함
    - 미래 날짜는 불가능
  - 이메일: 이메일 형식을 따라야 함
- **Main Flow**
  - Guest 가 아이디, 비밀번호, 이름, 생년월일, 이메일을 입력하고 회원가입을 시도한다.
  - 각 입력값이 위 조건을 만족하는지 확인한다.
  - 이상이 없으면 중복 ID를 확인한다.
  - 중복된 ID가 없으면 비밀번호를 암호화해서 회원정보를 저장한다.
  - 저장이 완료되면 회원가입을 완료한다.
- **Exception Flow**
  - 입력값이 조건을 만족하지 못하면 Conditions 나열 순서 기준으로 만족하지 못한 첫 번째 항목에 대해 사용자에게 알린다.
  - 중복된 ID가 있으면 회원가입에 실패하고 중복 ID가 존재함을 사용자에게 알린다.
- **After**
  - 회원가입이 완료되면 로그인 페이지로 이동

<br/>

### 2. 내 정보 조회
- **Actor**: User
- **Conditions**
  - 인증된 유저만 내 정보 조회가 가능하다.
- **Main Flow**
  - 유저가 내 정보 조회를 요청한다.
  - 해당 유저의 정보를 조회한다.
  - 이름의 마지막 글자를 '*' 로 마스킹한다.
  - 아이디, 마스킹 된 이름, 생년월일, 이메일을 유저에게 반환한다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.

<br/>


### 3. 비밀번호 변경
- **Actor**: User
- **Conditions** 
  - 인증된 유저만 비밀번호를 변경할 수 있다.
  - 변경할 비밀번호는 현재 비밀번호와 동일할 수 없다.
  - 비밀번호 기본 규칙은 [회원가입 비밀번호 조건](#signup-pw) 과 동일하다.
- **Main Flow**
  - 유저가 바꿀 비밀번호를 입력하고 비밀번호 변경을 요청한다.
  - 비밀번호가 기본 규칙을 만족하는지 확인한다.
  - 이상이 없으면 해당 유저의 기존 비밀번호를 조회한다.
  - 변경할 비밀번호가 기존 비밀번호와 다르면 비밀번호 변경해서 저장한다.
  - 사용자에게 비밀번호 변경이 완료됐음을 알린다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
  - 변경할 비밀번호가 기본 규칙을 따르지 않으면 올바르지 않은 비밀번호라고 사용자에게 알린다.
  - 변경할 비밀번호가 현재 비밀번호와 동일하면 동일한 비밀번호로 변경할 수 없음을 알린다.
- **After**
  - 비밀번호 변경이 완료되면 로그인 페이지로 이동한다.
<br/>


## 상품/브랜드
※ 관리자 권한 여부는 별도 인증을 통해 확인
### 1. 브랜드 등록
- **Actor**: Admin
- **Conditions**
  - 관리자만 브랜드를 등록할 수 있다.
  - 브랜드명, 브랜드 설명은 필수(빈값 불가)
  - 브랜드명은 중복될 수 없다.
- **Main Flow**
  - 관리자가 브랜드명, 브랜드 설명을 입력하고 브랜드 등록을 요청한다.
  - 브랜드명이 이미 존재하는지 확인한다.
  - 중복이 없으면 브랜드를 저장한다.
  - 저장이 완료되면 관리자에게 브랜드 등록 완료를 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
  - 필수 항목이 비어있으면 해당 항목이 필요함을 알린다.
  - 브랜드명이 이미 존재하면 중복된 브랜드명이 존재함을 알린다.

<br/>

### 2. 브랜드 수정
- **Actor**: Admin
- **Conditions**
  - 관리자만 브랜드 정보를 수정할 수 있다.
  - 브랜드 설명만 수정할 수 있다.(빈값 불가)
- **Main Flow**
  - 관리자가 브랜드를 선택, 설명을 입력하고 수정을 요청한다.
  - 해당 브랜드가 존재하면 설명을 수정하고 저장한다.
  - 저장이 완료되면 관리자에게 브랜드 수정 완료를 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
  - 필수 항목이 비어있으면 해당 항목이 필요함을 알린다.
  - 브랜드가 존재하지 않으면 브랜드가 존재하지 않음을 알린다.

<br/>

### 3. 브랜드 삭제
- **Actor**: Admin
- **Conditions**
  - 관리자만 브랜드를 삭제할 수 있다.
- **Main Flow**
  - 관리자가 브랜드를 선택하고 삭제를 요청한다.
  - 해당 브랜드에 속한 상품과 좋아요가 함께 삭제됨을 안내하고 재확인한다.
  - 확인되면 해당 브랜드가 존재하면 브랜드를 삭제한다.
  - 브랜드가 존재하지 않더라도 삭제 완료로 처리한다.
  - 관리자에게 브랜드 삭제 완료를 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
- **After**
  - 해당 브랜드에 속한 모든 상품이 함께 삭제된다.
  - 삭제된 상품에 대한 좋아요도 함께 삭제된다.

<br/>

### 4. 브랜드 목록 조회
- **Actor**: Admin
- **Conditions**
  - 관리자만 브랜드 목록을 조회할 수 있다.
- **Main Flow**
  - 관리자가 브랜드 목록 조회를 요청한다.
  - 등록된 전체 브랜드 목록(브랜드명)을 반환한다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.

<br/>

### 5. 브랜드 상세 조회
- **Actor**: Guest / User / Admin
- **Conditions**
  - 누구나 브랜드 상세 정보를 조회할 수 있다.
- **Main Flow**
  - 사용자가 브랜드를 선택하고 상세 조회를 요청한다.
  - 해당 브랜드의 상세 정보(브랜드명, 브랜드 설명)를 반환한다.
- **Exception Flow**
  - 브랜드가 존재하지 않으면 브랜드가 존재하지 않음을 알린다.

<br/>

### 6. 상품 등록
- **Actor**: Admin
- **Conditions**
  - 관리자만 상품을 등록할 수 있다.
  - 상품을 등록할 때는 이미 등록된 브랜드로만 등록할 수 있다.
  - 상품 등록 시에 '브랜드', '상품명', '가격', '상품설명'은 필수로 입력한다.
  - 가격은 원화 기준으로 입력하며 최소금액은 100원으로 제한한다.
  - 같은 브랜드 내에 동일한 상품명으로 등록할 수 없다.
  - 재고 수량은 선택 입력이며, 미입력 시 0으로 설정된다.
- **Main Flow**
  - 관리자가 브랜드를 선택하고 등록할 상품 정보(상품명, 가격, 상품설명, 재고)를 입력한다.
  - 필수 정보가 다 입력되면 해당 정보들을 저장한다.
  - 관리자에게 상품 등록이 완료되었음을 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
  - 필수정보 누락 시 등록에 실패하고 어떤 정보가 누락되었는지 알린다.
  - 상품을 등록하려는 브랜드가 존재하지 않는 경우 브랜드가 존재하지 않음을 알린다.
  - 같은 브랜드 내에 동일한 상품명이 존재하면 중복된 상품명이 존재함을 알린다.
- **After**
  - 상품 등록이 완료되면 상품을 등록한 브랜드의 상품 목록으로 이동
<br/>

### 7. 상품 정보 수정
- **Actor**: Admin
- **Conditions**
  - 관리자만 상품 정보를 수정할 수 있다.
  - 상품 가격, 상품 설명, 재고를 수정할 수 있다.(수정 시 세 항목 모두 전송)
- **Main Flow**
  - 관리자가 상품의 정보를 수정하고 저장을 요청한다.
  - 상품 가격이 100원 이상이고 상품 설명이 빈값이 아니며 재고가 0 이상이면 상품 정보를 수정한다.
  - 관리자에게 상품 정보가 수정되었음을 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
  - 수정하려는 상품이 존재하지 않는 경우 이미 상품이 삭제됐음을 알린다.
  - 가격이 100원 미만이거나 상품 설명이 빈값이거나 재고가 0 미만인 경우 수정에 실패하고 어떤 정보가 잘못되었는지 알린다.

- **After**
  - 수정된 상품 정보가 보일 수 있도록 상품 상세 정보로 이동

### 8. 상품 삭제
- **Actor**: Admin
- **Conditions**
  - 관리자만 상품을 삭제할 수 있다.
- **Main Flow**
  - 관리자가 상품을 선택하고 삭제를 요청한다.
  - 해당 상품에 대한 좋아요가 함께 삭제됨을 안내하고 재확인한다.
  - 확인되면 해당 상품이 존재하면 상품을 삭제한다.
  - 상품이 존재하지 않더라도 삭제 완료로 처리한다.
  - 관리자에게 상품 삭제 완료를 알린다.
- **Exception Flow**
  - 관리자 권한이 아닌 경우 접근할 수 없음을 알린다.
- **After**
  - 해당 상품에 대한 좋아요도 함께 삭제된다.
  - 상품 삭제가 완료되면 해당 브랜드의 상품 목록으로 이동

<br/>

### 9. 상품 목록 조회
- **Actor**: Guest / User / Admin
- **Conditions**
  - 누구나 상품 목록을 조회할 수 있다.
  - 브랜드별 필터링이 가능하다.
  - 최신 등록순(기본), 가격 낮은순, 좋아요 많은순으로 정렬이 가능하다.
  - 페이징을 적용한다.(기본 20개)
- **Main Flow**
  - 사용자가 상품 목록 조회를 요청한다.
  - 필터, 정렬, 페이징 조건에 따라 상품 목록(상품명, 가격, 브랜드명, 좋아요 수)을 반환한다.
- **Exception Flow**
  - 브랜드 필터링 시 해당 브랜드가 존재하지 않으면 브랜드가 존재하지 않음을 알린다.

<br/>

### 10. 상품 상세 조회
- **Actor**: Guest / User / Admin
- **Conditions**
  - 누구나 상품 상세 정보를 조회할 수 있다.
- **Main Flow**
  - 사용자가 상품을 선택하고 상세 조회를 요청한다.
  - 해당 상품의 상세 정보(상품명, 가격, 상품설명, 브랜드명, 좋아요 수)를 반환한다.
  - 관리자의 경우 재고 수량도 함께 반환한다.
- **Exception Flow**
  - 상품이 존재하지 않으면 상품이 존재하지 않음을 알린다.

<br/>

## 주문

### 1. 주문 요청
- **Actor**: User
- **Conditions**
  - 인증된 유저만 주문할 수 있다.
  - 하나 이상의 상품을 선택해야 한다.
  - 각 상품의 수량은 1개 이상이어야 한다.
  - 주문하려는 상품은 모두 존재하는 상품이어야 한다.
  - 주문하려는 각 상품의 재고가 주문 수량 이상이어야 한다.
- **Main Flow**
  - 유저가 상품과 수량을 선택하고 주문을 요청한다.
  - 선택한 상품이 모두 존재하는지 확인한다.
  - 이상이 없으면 각 상품의 재고가 충분한지 확인한다.
  - 재고가 충분하면 재고를 차감한다.
  - 주문 정보(주문 상품, 수량, 주문 시점의 상품명, 브랜드명, 상품 가격, 주문일시)를 저장한다.
  - 유저에게 주문이 완료되었음을 알린다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
  - 상품을 선택하지 않았거나 수량이 1개 미만이면 주문 정보가 올바르지 않음을 알린다.
  - 존재하지 않는 상품이 포함된 경우 해당 상품이 존재하지 않음을 알린다.
  - 주문 상품 중 하나라도 재고가 부족하면 주문에 실패하고 재고가 부족함을 알린다.
- **After**
  - 주문 완료 후 주문 상세 정보로 이동

<br/>

### 2. 유저의 주문 목록 조회
- **Actor**: User
- **Conditions**
  - 인증된 유저만 자신의 주문 목록을 조회할 수 있다.
  - 조회기간을 통해 특정 기간 동안의 주문을 조회할 수 있다. (미지정 시 최근 3개월)
- **Main Flow**
  - 유저가 주문 목록 조회를 요청한다.
  - 조회기간 조건에 해당하는 주문 목록(주문번호, 주문일시, 대표 상품명 외 N건, 총 금액)을 최신 주문순으로 반환한다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.

<br/>

### 3. 단일 주문 상세 조회
- **Actor**: User
- **Conditions**
  - 인증된 유저만 자신의 주문을 상세 조회할 수 있다.
- **Main Flow**
  - 유저가 주문을 선택하고 상세 조회를 요청한다.
  - 해당 주문의 상세 정보(주문번호, 주문일시, 주문 상품 목록(상품명, 브랜드명, 수량, 가격), 총 금액)를 반환한다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
  - 주문이 존재하지 않거나 본인의 주문이 아닌 경우 주문이 존재하지 않음을 알린다.

## 좋아요

### 1. 상품 좋아요 등록
- **Actor**: User
- **Conditions**
  - 인증된 유저만 좋아요를 등록할 수 있다.
  - 이미 좋아요한 상품에 다시 좋아요해도 성공으로 처리한다.
- **Main Flow**
  - 유저가 상품을 선택하고 좋아요를 요청한다.
  - 해당 상품이 존재하는지 확인한다.
  - 좋아요를 등록한다.(이미 좋아요한 경우 무시)
  - 유저에게 좋아요 등록이 완료되었음을 알린다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
  - 상품이 존재하지 않으면 상품이 존재하지 않음을 알린다.

<br/>

### 2. 상품 좋아요 취소
- **Actor**: User
- **Conditions**
  - 인증된 유저만 좋아요를 취소할 수 있다.
  - 좋아요하지 않은 상품에 취소해도 성공으로 처리한다.
- **Main Flow**
  - 유저가 상품을 선택하고 좋아요 취소를 요청한다.
  - 해당 상품이 존재하는지 확인한다.
  - 좋아요를 취소한다.(좋아요하지 않은 경우 무시)
  - 유저에게 좋아요 취소가 완료되었음을 알린다.
- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
  - 상품이 존재하지 않으면 상품이 존재하지 않음을 알린다.

<br/>

### 3. 내가 좋아요 한 상품 목록 조회
- **Actor**: User
- **Conditions**
  - 인증된 유저만 자신이 좋아요한 상품 목록을 조회할 수 있다.
- **Main Flow**
  - 유저가 좋아요한 상품 목록 조회를 요청한다.

- **Exception Flow**
  - 인증에 실패하는 경우 회원정보가 올바르지 않음을 알린다.
