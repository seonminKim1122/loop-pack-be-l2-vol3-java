## 요구사항 정의서

### 목적/범위
- 고객이 좋아하는 브랜드의 상품들을 한 번에 담아 주문할 수 있는 이커머스 서비스 만들기

### 액터
1. Guest: 회원가입 하지 않은 사용자
2. Member: 회원가입 한 사용자
3. Admin: 관리자


## 도메인 별 요구사항 정의
### 인증/인가
#### 1. 인증
   - 요청 헤더
     - X-Loopers-LoginId: 로그인ID
     - X-Loopers-LoginPw: 비밀번호
   - 헤더 정보를 이용해 회원 정보를 인증

#### 2. 인가
   - 요청 헤더
     - X-Loopers-Ldap: loopers.admin
   - 헤더 정보를 이용해 관리자 권한 확인

### 회원
#### 1. 회원가입
- Actor
  - Guest
- Constraints
  - 중복 로그인 ID로는 회원가입 불가
  - 로그인 ID, 비밀번호, 이름, 생년월일, 이메일 필수 입력
  - 로그인 ID
    - 영문과 숫자 6 ~ 20자만 허용
  - 비밀번호
    - 8 ~ 16자의 영문, 숫자, 특수문자만 가능
    - 생년월일은 비밀번호 내에 포함 불가
    - 암호화해서 저장
  - 이름
    - 한글 2 ~ 6자만 허용
  - 생년월일
    - 'yyyyMMdd' 양식만 허용
    - 미래 날짜는 불가
  - 이메일
    - 이메일 양식에 맞는 것만 허용
  
- Main Flow
  - Guest 가 로그인 ID, 비밀번호, 이름, 생년월일, 이메일을 입력하고 회원가입을 시도
  - 모든 입력값이 입력값에 대한 Constraints 를 충족하면 중복 ID 검사
  - 중복 ID가 존재하지 않으면 회원가입 성공
- Exception Flow
  - 회원가입 시 입력값에 대한 Constraints 를 충족하지 못하는 경우
    - 로그인 ID > 비밀번호 > 이름 > 생년월일 > 이메일 순으로 충족하지 못 하는 케이스에서 예외 발생
    - 예외 상황에 맞는 메시지 노출(ex: 비밀번호 양식이 올바르지 않습니다)
  - 중복 ID가 존재하는 경우
    - `중복된 ID가 존재합니다` 메시지 노출

#### 2. 내 정보 조회
- Actor
  - Member
- Constraints
  - 인증된 회원에 한해서만 내 정보 조회 가능
- Main Flow
  - Member 가 내 정보 조회 시도
  - [인증된 회원인지 확인](#1-인증)
  - 인증된 회원이면 해당 회원의 정보 조회
  - (로그인 ID, 이름, 생년월일, 이메일) 반환
  - 단, 이름 마지막 글자는 '*' 로 마스킹
- Exception Flow
  - 인증된 회원이 아닌 경우
    - `회원 정보가 올바르지 않습니다` 메시지 노출

#### 3. 비밀번호 수정
- Actor
  - Member
- Constraints
  - 인증된 회원에 한해서만 비밀번호 수정 가능
  - 비밀번호
    - [회원가입](#1-회원가입) 시 비밀번호 규칙을 따라야 함
    - 현재 비밀번호와 동일한 비밀번호는 사용 불가
- Main Flow
  - Member 가 새로운 비밀번호를 입력해 비밀번호 수정 시도
  - [인증된 회원인지 확인](#1-인증)
  - 인증된 회원이면 현재 회원 정보를 조회하고<br> 입력한 신규 비밀번호가 Constraints 를 충족하는지 확인
  - 충족하면 비밀번호 수정 성공
- Exception Flow
  - 인증된 회원이 아닌 경우
    - `회원 정보가 올바르지 않습니다` 메시지 노출
  - 비밀번호가 Constraints 를 충족하지 못하는 경우
    - `신규 비밀번호가 올바르지 않습니다` 메시지 노출


### 브랜드/상품
#### 1. 브랜드 등록
- Actor
  - Admin
- Constraints
  - 관리자 권한을 가진 경우에만 신규 브랜드 등록 가능
  - 브랜드명은 필수 입력
  - 중복된 브랜드명은 등록 불가
- Main Flow
  - Admin 이 브랜드명, 설명을 입력하고 신규 브랜드 등록 시도
  - [관리자 권한인지 확인](#2-인가)
  - 관리자 권한이면 브랜드명 중복을 확인하고 중복이 없으면 등록 성공
- Exception Flow
  - 관리자 권한이 아닌 경우
    - `관리자만 사용할 수 있는 기능입니다` 메시지 노출
  - 브랜드명이 누락된 경우
    - `브랜드명은 필수 입력사항입니다` 메시지 노출
  - 중복된 브랜드명이 있으면
    - `중복된 이름의 브랜드가 존재합니다` 메시지 노출

#### 2. 브랜드 정보 수정

  - Admin

- Main Flow
  - Admin 이 브랜드명과 설명을 입력하고 브랜드 정보 수정 시도
  - [관리자 권한인지 확인](#2-인가)
  - 관리자 권한이면 브랜드명 중복을 확인하고 중복이 없으면 수정 성공

#### 3. 등록된 브랜드 목록 조회
- Actor
  - Admin
- Constraints
  - 관리자 권한을 가진 경우에만 브랜드 목록 조회 가능
  - 페이징 기본 설정
    - page: 0
    - size: 20
  - 페이징 옵션 조건
    - page 는 0 또는 자연수만 가능
    - size 는 자연수 가능
- Main Flow
  - Admin 이 브랜드 목록 조회 시도
    - page, size 입력(선택사항)
    - 미 입력 시 기본 설정 적용
  - [관리자 권한인지 확인](#2-인가)
  - 관리자 권한이면 페이징 옵션이 Constraints 충족하는지 검사
  - 충족하면 등록일 기준 최신순으로 페이징 처리된 브랜드명 목록 조회해서 반환
- Exception Flow
  - 관리자 권한이 아닌 경우
    - `관리자만 사용할 수 있는 기능입니다` 메시지 노출
  - 페이징 옵션이 Constraints를 충족하지 못하거나 페이징 옵션에 맞는 브랜드 목록이 존재하지 않는 경우
    - `요청하신 페이지가 존재하지 않습니다` 메시지 노출

#### 8. 등록된 상품 목록 조회
- Actor
  - Guest, Member, Admin
- Constraints
  - 특정 브랜드에 속하는 상품만 별도 조회 가능
  - 등록일 최신순/낮은 가격순/좋아요 높은 순 조회 가능
  - 페이징 기본 설정
      - page: 0
      - size: 20
  - 페이징 옵션 조건
      - page 는 0 또는 자연수만 가능
      - size 는 자연수 가능
  - 정렬 기본 설정
    - sort: 등록일 최신순
- Main Flow
  - 사용자가 상품 목록 조회 시도
    - page, size, brandId, sort 는 선택 입력
    - page, size, sort 는 미 입력 시 기본 설정 반영
  - 페이징 옵션이 Constraints 충족하는지 검사
  - brandId 입력 시
    - 해당 브랜드가 존재하는지 검사
    - 존재하면 브랜드에 속하는 상품 중 sort, 페이징 옵션에 맞춰서 상품 목록 조회 및 상품 정보 목록(브랜드명, 상품명, 가격, 좋아요) 반환
  - brandId 미 입력 시
    - 전체 상품 중 sort, 페이징 옵션에 맞춰서 상품 목록 조회 및 상품 정보 목록(브랜드명, 상품명, 가격, 좋아요) 반환
- Exception Flow
  - 페이징 옵션이 Constraints 를 충족하지 못 하거나 페이징 옵션에 맞는 상품 목록이 없는 경우
    - `요청하신 페이지가 존재하지 않습니다` 메시지 노출
  - brandId 입력 시 해당 브랜드가 존재하지 않는 경우
    - `등록되지 않은 브랜드입니다` 메시지 노출
  - sort 옵션이 등록일 최신순/낮은 가격순/좋아요 높은 순 외에 다른 옵션인 경우
    - `정렬 기준이 올바르지 않습니다` 메시지 노출

  - [인증된 회원인지 확인](#1-인증)
  - 인증된 회원이면 해당 상품에 대한 좋아요 이력을 삭제
    - 상품이 존재하지 않거나 좋아요 이력이 존재하지 않아도 정상 삭제 처리
- Exception Flow

    - `회원 정보가 올바르지 않습니다` 메시지 노출

### 주문
#### 1. 주문 요청
- Actor
  - Member
- Constraints
  - 인증된 회원만 주문을 할 수 있다
  - 주문 수량은 자연수만 가능
- Main Flow
  - Member 가 원하는 상품들과 수량을 선택한 후 주문 시도
  - [인증된 회원인지 확인](#1-인증)
  - 인증된 회원이면 각 상품별로 존재여부 및 재고 확인
  - 상품이 존재하고 재고도 충분하면 정상 주문 처리
    - 주문 상품 정보는 주문 시점의 스냅샷 정보로 저장
- Exception Flow
  - 인증된 회원이 아닌 경우
    - `회원 정보가 올바르지 않습니다` 메시지 노출
  - 등록된 상품이 아닌 경우
    - `등록되지 않은 상품입니다` 메시지 노출
  - 주문 수량이 자연수가 아닌 경우
    - `주문 수량이 올바르지 않습니다` 메시지 노출
  - 상품 재고가 부족한 경우
    - ```
      재고가 부족한 상품이 포함되어 있습니다
      상품: A, 주문수량: 3, 재고수량: 1
      상품: B, 주문수량: 2, 재고수량: 0
      ...
      ```

#### 2. 주문 목록 조회(회원)
- Actor
  - Member
- Constraints
  - 인증된 회원만 주문 목록을 조회 가능
  - 조회하려는 주문기간을 입력하지 않는 경우 최근 6개월간 주문 목록 반환
  - 주문기간 조건
    - startAt: yyyy-MM-dd
    - endAt: yyyy-MM-dd
    - startAt <= endAt 
- Main Flow
  - Member 가 내 주문 목록 조회 시도
  - [인증된 회원인지 확인](#1-인증)
  - 인증된 회원이면 입력한 주문기간 내에 발생한 주문 목록 조회
    - 입력한 주문기간이 없으면 조회시점 기준 최근 6개월
  - 주문일자 최신순으로 주문 목록 반환(주문ID, 총 주문 금액, 상품명 목록)
- Exception Flow
  - 인증된 회원이 아닌 경우
    - `회원 정보가 올바르지 않습니다` 메시지 노출
  - 조회 주문기간이 형식을 따르지 않는 경우
    - `조회 주문기간은 yyyy-MM-dd 형식을 따라야 합니다` 메시지 노출
  - 조회 주문기간 시작일이 종료일보다 느린 경우
    - `시작일은 종료일 이전이어야 합니다` 메시지 노출

#### 3. 주문 목록 조회(관리자)
- Actor
  - Admin
- Constraints
  - 관리자 권한인 경우만 가능
  - 페이징 기본 설정
    - page: 0
    - size: 20
  - 페이징 옵션 조건
    - page: 0 또는 자연수
    - size: 자연수
- Main Flow
  - 관리자가 주문 목록 조회 시도
  - [관리자 권한인지 확인](#2-인가)
  - 관리자 권한이면 주문일자 최신순으로 페이징 옵션 적용해서 주문 목록 조회
  - 주문 목록 반환(주문ID, 총 주문 금액, 상품명 목록)
- Exception Flow
  - 관리자 권한이 아닌 경우
    - `관리자만 사용할 수 있는 기능입니다` 메시지 노출
  - 페이징 옵션이 조건을 충족하지 않는 경우
    - `요청하신 페이지가 존재하지 않습니다` 메시지 노출

  - 존재하지 않는 주문ID인 경우
    - `주문 정보가 존재하지 않습니다` 메시지 노출
